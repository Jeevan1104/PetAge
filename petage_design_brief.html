01 · Brand Essence

What PetAge Feels Like

PetAge sits at the intersection of a medical records app and a care companion. The design should feel like Apple Health crossed with a premium consumer brand — trusted, calm, and precise. Never playful, never clinical to the point of cold.

Trustworthy
Clean whitespace, medical-grade precision in data display. Every record feels permanent and retrievable.
Warm
Pet photos front and center. Rounded cards. Gentle status colors. The data is clinical; the presentation is human.
In Control
Owner-first. Every action is immediate. No waiting for vet approval. The owner owns their data — full stop.

10 · Dark Mode

Dark Mode Tokens

Required for App Store compliance (iOS Dark Mode)
All 18 screens need dark variants. Navy → near-black #0A0F1A. Cards → #1A2335. Primary blue shifts to #3B82C4 (lighter for contrast). Status colors use pastel tints on text, dark surfaces as backgrounds.

12 · Implementation Plan

14-Day Build Checklist

Web-first (Next.js 14), then mobile (Expo 51). 4–6 hrs/day. Each phase builds on the last — do not skip ahead. Check off items as you complete them.

Day 1
Phase 1 — Foundation & Infrastructure
Project setup · Firebase · Design system
1.1 Project Setup
  • Create Next.js 14: npx create-next-app@14 petage --typescript --tailwind --app
  • Initialize shadcn/ui: npx shadcn-ui@latest init
  • Install all TRD §3 deps: firebase, zustand, react-hook-form, zod, date-fns, recharts, @react-pdf/renderer, resend, stripe, @stripe/stripe-js, @upstash/ratelimit
  • TypeScript strict mode, ESLint, Prettier config
  • Create .env.local with all 15 env vars from TRD §6
1.2 Firebase Setup
  • Create Firebase project → enable Firestore (test mode), Auth (email/password + Google), Storage
  • Download Firebase config → populate env vars · Create service account JSON for Admin SDK
1.3 Design System
  • Implement full color palette into Tailwind config (all 10 colors from §02)
  • Configure Inter font (Google Fonts), weights 400/600 only
  • Build base components: Button (3 variants), Input, StatusPill, Card, FAB
Days 1–2
Phase 2 — Authentication
Screens 1–3 · Auth store · Route protection
  • Create useAuthStore (Zustand): login, logout, user state, tier tracking
  • Firebase Auth email/password sign up + sign in (v9 modular SDK)
  • Google SSO via GoogleAuthProvider
  • Create user doc in Firestore users collection on signup (exact TRD §4 schema)
  • Screen 1 (Splash), Screen 2 (Sign Up), Screen 3 (Log In) — react-hook-form + zod validation
  • Next.js middleware: check Firebase JWT on all /app/(dashboard)/* routes
  • Deploy to Vercel — verify auth works in production before continuing
Days 2–3
Phase 3 — Home Screen & Pet Profiles
Screens 4, 5, 15 · /api/pets endpoints
  • Screen 4 (Home): horizontal scrolling pet cards, "Add Pet" dashed card, bottom nav
  • Screen 5 (Pet Dashboard): photo header, 4 action tiles, activity feed, first-run tooltip
  • Screen 15 (Add Pet): photo → Firebase Storage, all fields per TRD schema
  • Wire /api/pets GET + POST + DELETE with Firestore security rules
  • Enforce 2-pet free limit server-side — block 3rd pet, show upgrade prompt
Stitch ref: af9f3b2a — PetAge Home Dashboard (Desktop 2560×2048)
Day 3
Phase 4 — Vaccine Tracker
Screens 6, 7 · Status calc · Reminder toggle
  • Screen 6 (Vaccine List): desc sort, status badges, red sticky "Action Required" header, FAB
  • Screen 7 (Add/Edit Vaccine): freeform name, date administered, expiry date, reminder toggle (default ON)
  • Status calc via date-fns: current / due_soon / overdue
  • Wire /api/vaccines CRUD — composite index: petId + ownerId + dateAdministered DESC
Stitch ref: b4958c10 — PetAge Vaccine List (Desktop 2560×2048)
Day 4
Phase 5 & 6 — Vet Visits + Medications
Screens 8–11 · Photo upload · Mark as Given

  • Screen 9 (Add/Edit Visit): all fields + <input type="file"> → Firebase Storage signed URL (1hr expiry)
  • Screen 10 (Med List): active meds with next-due countdown, archive button, always dismissable
  • Screen 11 (Add/Edit Med): freeform name (brand OR generic OR OTC), dosage strength, frequency, start date
  • Mark as Given: POST /api/medications/[id]/mark-given → recalculate nextDueDate, reset reminderSent

Phase 9 — Notifications & Reminders

13 · Critical Design Decisions

Why We Made These Choices

DecisionChoiceRationale
Build orderWeb firstFaster iteration; shared API means mobile is a UI port, not a rebuild
DatabaseFirebase Spark$0/month for 1,000 users; 50k reads/day + 20k writes/day is enough
PDF@react-pdf/rendererClient-side; no Lambda cost; generates as browser blob
Data ownershipFirestore: uid == ownerIdOwner-first enforced at DB level — cannot be bypassed via API
PaywallSoft (prompt, not hard block)Better UX; user explores before hitting gate; reduces day-1 bounce
Medication fieldsFreeform + dosageStrengthFixes Pawprint's #1 complaint — accepts generic, OTC, any name
Reminder dismissArchive + snooze always availableFixes Pawprint's undismissable reminder failure — owner controls all
EmailResend3,000 free/month; simplest API; reliable deliverability
Rate limitingUpstash Redis10,000 commands/day free; @upstash/ratelimit is AI-tool friendly
State managementZustandSame store logic shared across web + mobile; no Provider wrapping
14 · Verification Plan

Manual Verification Checklist

Run all 12 checks in production — not localhost, not staging. Every check must pass before App Store submission.

1
Auth: Sign up → sign in → Google SSO → logout → forgot password — all paths work on petage.app
2
Pet CRUD: Add 2 pets (free), attempt 3rd → upgrade prompt. Delete pet → list updates.
3
Vaccine: Add → verify status badge → edit expiry to past → confirm badge changes to "Overdue"
4
Vet visit: Add with 10MB photo → signed URL loads within 5 seconds
5
Medication: Add daily med → mark as given → verify nextDueDate = today + 1 day in Firestore
6
Weight: Log 5 entries → chart renders → toggle lbs/kg → values convert correctly
7
PDF gate: Free user sees lock overlay. Premium user downloads PDF with correct data.
8
Stripe: Complete checkout → tier = 'premium' in Firestore → cancel → tier reverts to 'free'
9
Notifications: Hit cron manually → Expo push received on device + Resend log shows sent
10
Security: User B cannot read User A's pets — direct Firestore query returns permission-denied
11
Responsive: Layout correct at 375 / 390 / 430 / 768 / 1280px — no overflow, no clipped text
12
Crash-free: Sentry / Crashlytics shows > 99% crash-free on first 100 sessions
